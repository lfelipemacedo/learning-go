# Projeto 04 — API REST de Catálogo

## Objetivo do projeto
Construir uma API REST para catálogo de produtos, praticando handlers, JSON, validação, middleware e contexto.

## O que este projeto exercita
- `net/http` e handlers;
- serialização e desserialização JSON;
- paginação e filtros;
- middleware de logging e recuperação;
- testes de integração básicos.

## Escopo inicial sugerido
- CRUD de produtos;
- paginação;
- filtros simples;
- validação de entrada;
- persistência inicial em memória.

## Como desenvolver
1. Comece pelo CRUD em memória.
2. Padronize requests e responses.
3. Adicione middleware e tratamento de erro.
4. Depois evolua para persistência real.

## Critérios de aceite
- a API responde com contratos consistentes;
- erros retornam status e mensagens adequadas;
- filtros e paginação funcionam como esperado;
- existe ao menos um teste de integração simples.
