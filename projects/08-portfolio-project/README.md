# Projeto 08 — Projeto Avançado de Portfólio

## Objetivo do projeto
Consolidar toda a trilha em um projeto mais completo, que demonstre domínio de Go em modelagem, testes, concorrência, persistência e operação.

## O que este projeto exercita
- integração dos módulos anteriores;
- arquitetura e organização de pacotes;
- testes unitários e de integração;
- concorrência aplicada a um problema real;
- prontidão para produção.

## Escopo inicial sugerido
- ingestão de dados ou eventos;
- processamento interno;
- persistência de resultados;
- exposição de consulta via API;
- observabilidade básica.

## Como desenvolver
1. Escolha um domínio que faça sentido para portfólio.
2. Implemente uma primeira entrega vertical simples.
3. Expanda gradualmente persistência, concorrência e operação.
4. Finalize com documentação técnica e retrospectiva dos trade-offs.

## Critérios de aceite
- o fluxo fim a fim funciona de forma confiável;
- a arquitetura está clara e documentada;
- há cobertura para regras centrais e fluxos principais;
- o projeto demonstra maturidade técnica além de apenas “funcionar”.

## Próximo passo
Depois deste projeto, revise toda a trilha e escolha os módulos que merecem implementação completa no repositório.
